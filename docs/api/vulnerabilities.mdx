# Vulnerabilities

## List Vulnerabilities

Get a paginated list of vulnerabilities with optional filtering by workspace, severity, confidence, asset value, or lifecycle status.

**List all vulnerabilities:**
```bash
curl http://localhost:8002/osm/api/vulnerabilities \
  -H "Authorization: Bearer $TOKEN"
```

**List vulnerabilities with pagination:**
```bash
curl "http://localhost:8002/osm/api/vulnerabilities?offset=0&limit=100" \
  -H "Authorization: Bearer $TOKEN"
```

**Filter by workspace:**
```bash
curl "http://localhost:8002/osm/api/vulnerabilities?workspace=example.com" \
  -H "Authorization: Bearer $TOKEN"
```

**Filter by severity:**
```bash
curl "http://localhost:8002/osm/api/vulnerabilities?severity=critical" \
  -H "Authorization: Bearer $TOKEN"
```

**Filter by confidence:**
```bash
curl "http://localhost:8002/osm/api/vulnerabilities?confidence=Certain" \
  -H "Authorization: Bearer $TOKEN"
```

**Filter by asset value (partial match):**
```bash
curl "http://localhost:8002/osm/api/vulnerabilities?asset_value=api.example" \
  -H "Authorization: Bearer $TOKEN"
```

**Combine filters:**
```bash
curl "http://localhost:8002/osm/api/vulnerabilities?workspace=example.com&severity=high&offset=0&limit=50" \
  -H "Authorization: Bearer $TOKEN"
```

**Filter by lifecycle status:**
```bash
curl "http://localhost:8002/osm/api/vulnerabilities?workspace=example.com&status=retest" \
  -H "Authorization: Bearer $TOKEN"
```

**Query Parameters:**

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `workspace` | string | - | Filter by workspace name |
| `severity` | string | - | Filter by severity (critical, high, medium, low, info) |
| `confidence` | string | - | Filter by confidence (Certain, Firm, Tentative, Manual Review Required) |
| `asset_value` | string | - | Filter by asset value (partial match) |
| `status` | string | - | Filter by lifecycle status (`new`, `triaged`, `verified`, `false_positive`, `retest`, `closed`) |
| `fingerprint_key` | string | - | Filter by the merged fingerprint key |
| `source_run_uuid` | string | - | Filter by the currently linked source run UUID |
| `offset` | int | 0 | Number of records to skip |
| `limit` | int | 20 | Maximum records to return (max 10000) |

## Get Vulnerability Summary

Get a summary of vulnerabilities grouped by severity, optionally filtered by workspace.

**Get summary for all workspaces:**
```bash
curl http://localhost:8002/osm/api/vulnerabilities/summary \
  -H "Authorization: Bearer $TOKEN"
```

**Get summary for a specific workspace:**
```bash
curl "http://localhost:8002/osm/api/vulnerabilities/summary?workspace=example.com" \
  -H "Authorization: Bearer $TOKEN"
```

**Query Parameters:**

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `workspace` | string | - | Filter by workspace name |

**Response:**
```json
{
  "data": {
    "by_severity": {
      "critical": 2,
      "high": 5,
      "medium": 8,
      "low": 12,
      "info": 3
    },
    "total": 30,
    "workspace": "example.com"
  }
}
```

- `related_attack_chains`: matching attack-chain workbench entries resolved by ref, URL, or vulnerability name.

## Create Vulnerability

Create a new vulnerability record.

```bash
curl -X POST http://localhost:8002/osm/api/vulnerabilities \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "workspace": "example.com",
    "vuln_info": "CVE-2024-9999",
    "vuln_title": "Remote Code Execution",
    "vuln_desc": "Critical RCE vulnerability in admin panel.",
    "vuln_poc": "curl -X POST /admin/exec -d \"cmd=id\"",
    "severity": "critical",
    "asset_type": "web",
    "asset_value": "https://example.com/admin",
    "tags": ["rce", "critical", "admin"],
    "detail_http_request": "POST /admin/exec HTTP/1.1\n...",
    "detail_http_response": "HTTP/1.1 200 OK\nuid=0(root)...",
    "raw_vuln_json": "{\"template\":\"rce-admin.yaml\"}"
  }'
```

**Request Body:**

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `workspace` | string | Yes | Workspace/target name |
| `vuln_info` | string | No | CVE or vulnerability identifier |
| `vuln_title` | string | No | Short title for the vulnerability |
| `vuln_desc` | string | No | Detailed description |
| `vuln_poc` | string | No | Proof of concept |
| `severity` | string | No | Severity level (critical, high, medium, low, info) |
| `confidence` | string | No | Confidence level (Certain, Firm, Tentative, Manual Review Required) |
| `asset_type` | string | No | Type of asset (web, api, network, etc.) |
| `asset_value` | string | No | Affected asset URL or identifier |
| `tags` | array | No | Tags for categorization |
| `detail_http_request` | string | No | Raw HTTP request |
| `detail_http_response` | string | No | Raw HTTP response |
| `raw_vuln_json` | string | No | Raw JSON from scanner (nuclei, etc.) |
| `vuln_status` | string | No | Initial lifecycle status |
| `source_run_uuid` | string | No | Source run UUID used for dedup/timeline |
| `attack_chain_ref` | string | No | Optional reverse link to an attack-chain item |
| `related_assets` | array | No | Extra asset identifiers related to the finding |
| `report_refs` | array | No | Related report file references |

**Response (201 Created):**
```json
{
  "data": {
    "id": 15,
    "workspace": "example.com",
    "vuln_info": "CVE-2024-9999",
    "vuln_title": "Remote Code Execution",
    "vuln_desc": "Critical RCE vulnerability in admin panel.",
    "vuln_poc": "curl -X POST /admin/exec -d \"cmd=id\"",
    "severity": "critical",
    "confidence": "Certain",
    "asset_type": "web",
    "asset_value": "https://example.com/admin",
    "tags": ["rce", "critical", "admin"],
    "detail_http_request": "POST /admin/exec HTTP/1.1\n...",
    "detail_http_response": "HTTP/1.1 200 OK\nuid=0(root)...",
    "raw_vuln_json": "{\"template\":\"rce-admin.yaml\"}",
    "created_at": "2025-01-15T14:25:00Z",
    "updated_at": "2025-01-15T14:25:00Z"
  },
  "message": "Vulnerability created successfully"
}
```

If the submitted finding resolves to the same `workspace + fingerprint_key`, the API merges it into the existing row instead of creating a duplicate. In that case the endpoint returns `200 OK` with `merged: true`, `created: false`, and an incremented `evidence_version`.

## Update Vulnerability

Update lifecycle status, AI verdicts, analyst notes, and other editable fields.

```bash
curl -X PATCH http://localhost:8002/osm/api/vulnerabilities/15 \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "vuln_status": "verified",
    "ai_verdict": "confirmed",
    "analyst_notes": "Validated manually against staging"
  }'
```

Common lifecycle values:

- `new`
- `triaged`
- `verified`
- `false_positive`
- `retest`
- `closed`

## Queue a Retest Run

Queue a workflow or module to re-check a vulnerability.

```bash
curl -X POST http://localhost:8002/osm/api/vulnerabilities/15/retest \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "flow": "superdomain-extensive-ai-hybrid",
    "priority": "high",
    "params": {
      "recheck_mode": "focused"
    }
  }'
```

**Request Body**

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `flow` | string | No* | Flow to run for retest |
| `module` | string | No* | Module to run for retest |
| `priority` | string | No | Queue priority |
| `params` | object | No | Extra workflow params |

*One of `flow` or `module` is required.

**Response**

```json
{
  "message": "Retest task queued",
  "run_uuid": "retest-run-uuid"
}
```

**Error Response (400):**
```json
{
  "error": true,
  "message": "Workspace is required"
}
```

## Vulnerability Fields Reference

| Field | Type | Description |
|-------|------|-------------|
| `id` | int | Unique vulnerability identifier |
| `workspace` | string | Workspace/scan target name |
| `vuln_info` | string | CVE or vulnerability identifier |
| `vuln_title` | string | Short descriptive title |
| `vuln_desc` | string | Detailed vulnerability description |
| `vuln_poc` | string | Proof of concept exploit |
| `severity` | string | Severity level (critical, high, medium, low, info) |
| `confidence` | string | Confidence level (Certain, Firm, Tentative, Manual Review Required) |
| `asset_type` | string | Type of affected asset |
| `asset_value` | string | Affected asset URL or identifier |
| `tags` | array | Categorization tags |
| `detail_http_request` | string | Raw HTTP request that triggered the vulnerability |
| `detail_http_response` | string | Raw HTTP response from the vulnerable endpoint |
| `raw_vuln_json` | string | Raw JSON output from vulnerability scanner |
| `fingerprint_key` | string | Stable deduplication key derived from workspace, finding identity, asset type, and asset value |
| `vuln_status` | string | Lifecycle status |
| `source_run_uuid` | string | Source run UUID currently associated with the record |
| `retest_status` | string | Retest queue/execution state |
| `retest_run_uuid` | string | Run UUID of the queued retest |
| `attack_chain_ref` | string | Optional reverse link to an attack-chain record |
| `related_assets` | array | Related asset identifiers stored on the vulnerability |
| `report_refs` | array | Related report paths or references |
| `evidence_version` | int | Monotonic version number for evidence changes |
| `evidence_history` | string | Stored JSON history of evidence snapshots |
| `evidence_timeline` | array | Parsed evidence history returned by `GET /vulnerabilities/:id` |
| `status_timeline` | array | Version-aware status progression returned by `GET /vulnerabilities/:id` |
| `retest_timeline` | array | Retest-only run timeline returned by `GET /vulnerabilities/:id` |
| `related_runs` | array | Resolved run summaries returned by `GET /vulnerabilities/:id` |
| `related_asset_records` | array | Resolved asset rows returned by `GET /vulnerabilities/:id` |
| `related_attack_chains` | array | Resolved attack-chain links returned by `GET /vulnerabilities/:id` |
| `first_seen_at` | timestamp | First observed timestamp across merged evidence |
| `last_seen_at` | timestamp | Last evidence observation timestamp |
| `verified_at` | timestamp | Timestamp when the finding entered `verified` |
| `closed_at` | timestamp | Timestamp when the finding entered `closed` |
| `created_at` | timestamp | Record creation timestamp |
| `updated_at` | timestamp | Last update timestamp |
